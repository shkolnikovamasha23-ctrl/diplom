# Тест-план

## Объект тестирования
Веб-сайт EduSim: публичные страницы, авторизация, личный кабинет, симулятор, админ-панель, файловое хранилище, генерация документов.

## Подходы
Выбраны позитивное, негативное, функциональное, UI, инсталляционное и регрессионное тестирование.

## Критерии начала
Приложение собирается через Docker, база инициализируется, доступны демо-пользователи.

## Критерии окончания
Критические сценарии выполняются без ошибок: вход, прохождение симулятора, загрузка файла, генерация документов, просмотр админ-панели.

## Тест-кейсы
1. Открыть главную страницу: ожидается загрузка публичной страницы и меню.
2. Войти как student/student123: ожидается переход в личный кабинет.
3. Пройти сценарий: ожидается создание попытки и начисление баллов.
4. Скачать DOCX: ожидается файл отчета по попытке.
5. Скачать XLSX: ожидается выгрузка всех попыток.
6. Загрузить файл: ожидается запись в `uploaded_files` и файл в каталоге uploads.
7. Войти как admin/admin123: ожидается доступ к панели администратора.
8. Открыть `/admin` без прав: ожидается 403 или редирект на вход.
