# План тестирования

## Функциональное тестирование

1. Регистрация и вход пользователя.
2. Запуск сценария из личного кабинета.
3. Переход между этапами после выбора действия.
4. Изменение состояния симуляции: время, бюджет, риск, качество, регламент.
5. Блокировка перехода, если обязательный файл не загружен.
6. Сохранение файла с привязкой к пользователю, попытке и этапу.
7. Формирование истории решений и итогового отчета.
8. Отображение результатов в личном кабинете.

## Пользовательское тестирование

Для образовательного продукта добавлена проверка на реальных пользователях. Минимальная группа: 5 студентов. Им предлагается пройти сценарий, после чего заполнить короткую анкету.

Критерии UX:

- понятность вводной ситуации;
- понятность вариантов действий;
- ясность последствий выбора;
- удобство загрузки файла;
- понятность итогового отчета;
- общая оценка интерфейса по шкале 1–5.

## Проверяемая гипотеза

Если сценарий содержит контекст, последствия и итоговый отчет, студент лучше понимает причинно-следственную связь между решением и результатом, чем при прохождении обычного теста.
