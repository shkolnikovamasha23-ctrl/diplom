# Уточненная трассировка требований

## Файловая система

Загрузка файлов нужна не для аватарок и не как отдельное хранилище. В проекте файлы используются как артефакты прохождения симуляции:

- техническое задание или перечень требований на этапе сбора требований;
- отчет об ошибке или скриншот на этапе инцидента;
- дополнительные материалы студента для преподавателя;
- основание для итогового отчета по попытке.

Файл связывается с пользователем, попыткой прохождения и конкретным этапом сценария. Если этап помечен как обязательный для загрузки файла, пользователь не может перейти дальше без файла.

## Отличие симулятора от теста

Обычный тест проверяет правильность ответа на независимые вопросы. Образовательный симулятор моделирует ситуацию: у него есть состояние, этапы, варианты действий, последствия и итог. В EduSim состояние включает время, бюджет, риск, качество и соблюдение регламента. Выбор пользователя меняет эти показатели и определяет следующий этап.

## Сценарии

Сценарий состоит из:

- вводной ситуации;
- этапов сценария;
- вариантов действий;
- переходов между этапами;
- файловых артефактов;
- истории действий;
- итоговой оценки и текстового заключения.

## Оценка

Итоговая оценка не сводится к сумме правильных ответов. Учитываются:

- баллы за выбранные действия;
- затраченное время;
- бюджет;
- уровень риска;
- качество результата;
- соблюдение регламента;
- наличие обязательных файлов.
