# Бизнес-процессы симулятора

## Прохождение сценария

1. Студент выбирает сценарий.
2. Система создает попытку и начальное состояние.
3. Студент читает ситуацию и выбирает действие.
4. Система пересчитывает показатели: время, бюджет, риск, качество и регламент.
5. Если этап требует файл, студент загружает артефакт решения.
6. Система определяет следующий этап по выбранному действию.
7. После финального этапа формируется итоговое заключение.

## Работа с файлами

Файл является частью учебного действия. Например, студент должен загрузить ТЗ перед пилотной эксплуатацией или отчет о проблеме при инциденте. Это делает сценарий ближе к реальному процессу, где решение подтверждается документом.

## Экономическое обоснование

Расчет эффективности не строится только на условной ставке. Для оценки учитываются:

- разработка первой версии;
- подготовка сценариев;
- тестирование;
- сопровождение;
- обновление учебных материалов;
- хостинг.

Сравнение с готовой LMS ограничено: LMS закрывает хранение курсов и тестов, но не заменяет предметный сценарный симулятор с ветвлением, состоянием и файлами-артефактами.
